# SEO Guidelines

Use these as defaults unless the user provides better project-specific rules.

## Search Intent

- Match the likely intent before optimizing for keywords.
- Do not force transactional CTAs into purely informational articles.

## Titles

- Make the title specific.
- Include the primary keyword naturally when appropriate.
- Avoid obvious clickbait.

## Meta Descriptions

- Keep them concise.
- Promise a concrete benefit.
- Avoid repeating the title word-for-word.

## Structure

- Use one clear H1.
- Use H2s for major sections.
- Use H3s only when they improve clarity.
- Keep the article easy to scan.

## Content Quality

- Use concrete examples.
- Explain jargon when needed.
- Favor practical recommendations over filler.
- Add comparison language only when it helps the user decide.

## Linking

- Add internal links only when they genuinely help.
- Link external sources for claims, stats, or definitions when needed.
